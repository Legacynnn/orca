---
name: Think before coding
description: Read context and plan the change before touching any files.
---

Before you write any code for this task:

1. Read the relevant files end-to-end so you understand the existing patterns, naming, and abstractions. Don't skim.
2. State, in your own words, what the change needs to do and why — including which constraints, invariants, or callers it must respect.
3. Sketch the smallest correct change. Identify the files you'll touch and the order. Flag anything that needs a decision from me before you start.
4. Only then begin editing.

If, halfway through, you realise the plan was wrong, stop and re-plan rather than patching around the mismatch.
